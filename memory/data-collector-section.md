# Data Collector section (top-menu) — Jun 2026

Clicking the **Data Collector** top-bar item swaps the left sidebar to the DC menu
and opens list + edit pages. Built from `console-frontend` (the UI truth) and the
real DC microservices.

## Service ownership (confirmed via console-frontend api clients)
- **data-collector-api** → Projects, Tags, Templates, Output schemas, Data types
- **orders-management-api** → Orders, Executions, Tasks, **Error indicators**
- **data-collector-proxies-api** → Proxy providers, Proxy types, Proxy accounts
- `data-collector-instructions-api` = stateless converter (DTOs only, owns nothing).
- `data-collector-dispatcher` = Python K8s job launcher (owns nothing).
- `ecometry-tasks-api` = the *old* tasks model (Project/Job/Seed/Box/Timeframe) — NOT the DC backend; that area is the deprecating "Tasks" menu under Ecometry.

## Real enums (verbatim — used in DC_SPECS)
- Project status: ACTIVE, INACTIVE, ARCHIVED
- Extraction (DeliveryMethod.Stream): AD, FSA_CITY, FSA_MENU, FSA_RESTAURANT, MARKETPLACE, MEDIA, PDP, PDP_MARKETPLACE, PDP_REVIEWS, PLP, QCA_PLP, SEARCH, SHELF, GEN_CHAT, TEST_EXTRACTION
- Delivery method (sealed): None, Firehose (stream), S3 (bucket+folder), Rabbitmq (key+exchange)
- Execution status: PENDING, EMPTY, STARTED, CANCELLED, ERROR, FINISHED, HANGED
- Execution mode: MANUAL, AUTO · type: REPROCESSING, FULL, RE_EXECUTION · groupId batches runs
- Task status: PENDING, STARTED, UNPROCESSED, ERROR, WARNING, SUCCESS, CANCELLED, VOID (final = ERROR/WARNING/SUCCESS/VOID)
- Error category: SEED_ERROR, CONNECTION_ERROR, TEMPLATE_ERROR, DATA_COLLECTOR_ERROR, PROXY_ERROR, UNCLASSIFIED
- Machine size: XXS, XS, S, M, L, XL, XXL, Boost, Custom
- Re-execution rules: retries 1–5, next-trigger delay 0/15/30/45/60 min, error categories (excl UNCLASSIFIED)
- cacheValidity: 0–86400 s

## Key rules
- Order name unique among non-archived; can't archive+activate at once; system limit on scheduled orders.
- Execution can receive tasks only while none generated; details required on ERROR/HANGED; HANGED→STARTED/FINISHED.
- Task id is sequential per execution (key = id+executionId); CANCELLED can be re-queued.
- Proxy account/provider/type: name+slug unique per scope (slug lowercase, no spaces); account's type must belong to same provider; both must be active; archive forces inactive; provider can't be deactivated while it has active accounts; limitedRegions = ISO codes (es, gb, br-sp).
- Error indicator regex → category; `allowAutomaticTasksReExecution` gates eligibility; order re-execution rules pick which categories actually retry.
