# IAM section (top-menu) — Jun 2026

Clicking the **IAM** top-bar item swaps the left sidebar to the IAM menu (flat,
4 items) and opens list + edit pages. Grounded in `console-frontend/src/features/iam/*`
(no separate IAM backend repo is cloned — console-frontend is the source of truth).

## Menu (matches console.v2.shalion.com/iam screenshot)
Applications · Users · Accounts · Roles / Persona  — flat links, no groups.

## Entities (IAM_SPECS in src/lib/iamEntities.ts)
- **Applications** `/iam/applications` — Name, Slug, Created/Updated. Add "Add application". The 3 platform apps: Ecometry, Data Collector, IAM. Each owns permissions = resource:action pairs. Slug read-only once set.
- **Users** `/iam/users` — Email, Accounts, Role, Status, Created/Updated. **No Add button** (created via a batch flow). Email read-only; status Active/Inactive switch; one role per account.
- **Accounts** `/iam/accounts` — Name, Slug, Created/Updated. Add "Add account". Client/org context users belong to; has app clients (PUBLIC/CONFIDENTIAL); create/delete needs ADMIN or iam:account:manage.
- **Roles / Persona** `/iam/roles` — Name, Slug, Created/Updated. Add "Add role". Bundle of access assigned to a user within an account; slug auto + immutable; role:manage gates changes.

## Implementation (same pattern as Data Collector)
- Menu: `Sidebar.tsx` → `navForPath()` returns `iamNav` when pathname starts with `/iam`.
- Specs: `src/lib/iamEntities.ts` (IAM_SPECS + IAM_ROUTES), fed to the generic EntityListPage/EntityEditPage.
- Routes: folder routes under `src/routes/iam/**` (index/$id/new; Users has no `new`), generated by `/tmp/gen-iam-routes.mjs`. Removed the `iam.tsx` placeholder leaf; `iam/index.tsx` is the overview landing.
- Rules: new "IAM" RuleSection in `src/lib/businessRules.ts`; `?` manual works on every IAM page.
- Persistence keys are `approx:iam-*:v2` (the global v2 bump).
